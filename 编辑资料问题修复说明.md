# 编辑资料问题修复说明

## 问题分析

用户反馈编辑资料功能存在两个主要问题：

1. **获取用户信息失败：用户不存在**
2. **编辑资料保存后后端没有变化**

## 根本原因

### 问题1：用户不存在错误
**原因**：JWT认证和用户查找逻辑不匹配
- 登录时JWT使用**手机号**作为subject
- 认证时UserDetailsService使用**手机号**查找用户
- 但是AuthService.getCurrentUser()错误地使用了`userMapper.selectByUsername()`而不是`userMapper.selectByPhone()`

### 问题2：编辑资料后端无变化
**原因**：在修复页面跳转问题时，为了简化代码，临时移除了API调用，只做了本地数据更新

## 修复内容

### 1. 修复用户信息获取问题

**修改文件**：`backend/src/main/java/com/property/service/AuthService.java`

```java
// 修复前
String username = authentication.getName();
SysUser user = userMapper.selectByUsername(username); // 错误：用户名查找

// 修复后  
String phone = authentication.getName(); // JWT中存储的是手机号
SysUser user = userMapper.selectByPhone(phone); // 正确：手机号查找
```

### 2. 完善UserInfoDTO支持新字段

**修改文件**：`backend/src/main/java/com/property/dto/UserInfoDTO.java`

添加新字段：
- gender (性别)
- birthday (生日)
- signature (个人签名)
- emergencyContact (紧急联系人)
- emergencyPhone (紧急联系电话)

并添加对应的getter/setter方法。

### 3. 恢复API调用功能

**修改文件**：`uniapp/pages/owner/edit-profile.js`

恢复完整的API对接逻辑：
```javascript
// 调用API更新用户信息
api.updateProfile({
  realName: form.realName,
  phone: form.phone,
  email: form.email,
  gender: form.gender,
  birthday: form.birthday,
  signature: form.signature,
  emergencyContact: form.emergencyContact,
  emergencyPhone: form.emergencyPhone
})
```

### 4. 数据库字段更新

需要执行以下SQL脚本之一：
- `backend/add_user_profile_fields.sql` (带检查的完整版本)
- `backend/update_user_fields_simple.sql` (简化版本)

添加字段：
```sql
ALTER TABLE sys_user ADD COLUMN gender VARCHAR(10) COMMENT '性别';
ALTER TABLE sys_user ADD COLUMN birthday VARCHAR(20) COMMENT '生日';  
ALTER TABLE sys_user ADD COLUMN signature VARCHAR(500) COMMENT '个人签名';
ALTER TABLE sys_user ADD COLUMN emergency_contact VARCHAR(50) COMMENT '紧急联系人';
ALTER TABLE sys_user ADD COLUMN emergency_phone VARCHAR(20) COMMENT '紧急联系电话';
```

### 5. 移除调试代码

清理了profile.js中的临时调试输出，保持代码整洁。


2. **重启后端服务**：确保修改的Java代码生效

3. **测试功能**：

   - 测试编辑资料页面是否能正常跳转和加载
   - 测试保存功能是否能正确更新后端数据

## 技术要点

1. **JWT认证一致性**：确保JWT生成、解析、用户查找全程使用相同的标识符(手机号)
2. **数据传输完整性**：前后端数据结构保持一致，包含所有必要字段
3. **API错误处理**：完善的错误提示和降级方案
4. **数据库兼容性**：提供安全的迁移脚本，支持重复执行

3. 重新加载个人中心页面，检查修改的信息是否正确显示
