# 故障排除指南

## Swagger兼容性问题解决方案

### 问题描述
```
Failed to start bean 'documentationPluginsBootstrapper'; 
nested exception is java.lang.NullPointerException: 
Cannot invoke "org.springframework.web.servlet.mvc.condition.PatternsRequestCondition.getPatterns()" 
because "this.condition" is null
```

### 解决方案

#### 方案1：使用修复后的配置（推荐）
我已经在application.yml中添加了以下配置来解决兼容性问题：
```yaml
spring:
  mvc:
    pathmatch:
      matching-strategy: ant_path_matcher
```

并且将Swagger配置从OpenAPI 3.0改为Swagger 2.0：
```java
@EnableSwagger2  // 而不是 @EnableOpenApi
DocumentationType.SWAGGER_2  // 而不是 DocumentationType.OAS_30
```

#### 方案2：禁用Swagger启动（备用方案）
如果方案1仍有问题，可以使用无Swagger模式启动：

```bash
# 使用无Swagger启动脚本
start-no-swagger.bat

# 或者手动指定配置文件
mvn spring-boot:run -Dspring-boot.run.profiles=no-swagger
```

#### 方案3：测试系统是否正常运行
启动后访问测试接口验证系统功能：

```bash
# 健康检查
curl http://localhost:8081/api/test/health

# 系统信息
curl http://localhost:8081/api/test/info
```

### 启动步骤

1. **首先尝试正常启动**：
   ```bash
   start.bat
   ```

2. **如果遇到Swagger错误，使用无Swagger模式**：
   ```bash
   start-no-swagger.bat
   ```

3. **验证系统运行**：
   - 访问：http://localhost:8081/api/test/health
   - 应该返回：`{"code":200,"message":"系统运行正常",...}`

4. **测试登录功能**：
   ```bash
   curl -X POST http://localhost:8081/api/auth/login \
   -H "Content-Type: application/json" \
   -d '{"username":"admin","password":"123456"}'
   ```

### 访问地址

- **API基础地址**：http://localhost:8081/api
- **健康检查**：http://localhost:8081/api/test/health
- **系统信息**：http://localhost:8081/api/test/info
- **Druid监控**：http://localhost:8081/api/druid/index.html
- **Swagger文档**：http://localhost:8081/api/swagger-ui.html （如果启用）

### 数据库连接检查

如果遇到数据库连接问题：

1. **确认MySQL服务运行**：
   ```bash
   # Windows
   net start mysql
   
   # 或检查服务状态
   sc query mysql
   ```

2. **执行数据库初始化**：
   ```sql
   SOURCE src/main/resources/sql/init_database.sql;
   ```

3. **检查数据库配置**：
   - 用户名：root
   - 密码：123456（请修改为你的实际密码）
   - 数据库：property_management
