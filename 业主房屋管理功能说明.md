# 业主房屋管理功能说明

## 功能概述

重新设计了业主房屋管理功能，明确区分业主和租户的角色，业主只能查看和管理自己拥有的房产。

## 主要改进

### 1. 数据模型优化
- 使用 `owner_room` 关系表管理用户与房屋的关系
- 区分业主(OWNER)和租户(TENANT)两种关系类型
- 业主只能看到自己拥有的房产，不会看到其他房屋

### 2. 状态重新定义
**原有状态（不合理）：**
- VACANT（空置）
- OCCUPIED（已入住）
- RENTED（出租）

**新的状态（业主视角）：**
- SELF_OCCUPIED（自住）
- RENTED_OUT（出租中）
- VACANT（空置）

### 3. 权限控制
- 业主只能查看自己拥有的房屋
- 只能编辑自己房屋的面积和使用状态
- 验证房屋所有权后才允许操作

## 功能特性

### 1. 房屋列表显示
- 显示业主名下所有房产
- 展示房屋地址、楼层、面积、物业费等信息
- 根据房屋当前状态显示对应标签

### 2. 房屋信息编辑
- 可编辑房屋面积
- 可修改房屋使用状态（自住/出租中/空置）
- 数据验证和权限检查

### 3. 响应式设计
- 适配不同屏幕尺寸
- 优雅的卡片式布局
- 直观的状态颜色标识

## 技术实现

### 后端API
- `/owner/my-rooms` - 获取当前用户房屋
- `/owner/rooms/{roomId}` - 更新房屋信息
- `/owner/rooms/detail/{roomId}` - 查看房屋详情

### 数据库表
```sql
-- 用户房产关系表
CREATE TABLE owner_room (
  id bigint PRIMARY KEY AUTO_INCREMENT,
  user_id bigint NOT NULL COMMENT '用户ID',
  room_id bigint NOT NULL COMMENT '房屋ID', 
  relation_type varchar(20) NOT NULL COMMENT 'OWNER/TENANT',
  start_date date NOT NULL,
  end_date date DEFAULT NULL,
  status tinyint DEFAULT 1,
  ...
);
```

### 前端页面
- `pages/owner/house-info` - 房屋信息管理页面
- 集成下拉刷新、数据验证、错误处理
- 模态框编辑功能

## 使用说明

1. **查看房屋**：进入"我的房屋"页面，查看名下所有房产
2. **编辑信息**：点击"编辑信息"按钮，可修改面积和使用状态
3. **查看详情**：点击"查看详情"查看房屋完整信息
