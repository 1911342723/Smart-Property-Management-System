# 报修功能测试指南

## 🎯 功能概述

报修功能是物业管理系统的核心功能，包含工单提交、跟踪、处理、评价的完整流程。

## 📋 测试环境准备

### 1. 后端服务启动
```bash
cd backend
start.bat
```
确保后端服务在 `http://localhost:8081/api` 正常运行。

### 2. 数据库初始化
确保MySQL数据库已创建并执行了初始化脚本：
```sql
SOURCE src/main/resources/sql/init_database.sql;
```

### 3. 测试账号
| 角色 | 用户名 | 密码 | 用途 |
|------|--------|------|------|
| 业主 | owner001 | 123456 | 提交报修、查看进度、评价 |
| 维修工 | worker001 | 123456 | 接单、处理工单 |
| 管理员 | admin | 123456 | 分配工单、管理系统 |

## 🧪 完整测试流程

### 第一步：业主登录并提交报修

1. **启动小程序**
   - 打开微信开发者工具
   - 加载uniapp项目
   - 启动小程序

2. **用户登录**
   - 进入登录页面
   - 选择"业主"角色
   - 输入账号：owner001 / 123456
   - 点击登录

3. **提交报修**
   - 点击底部"报修"tab
   - 选择报修类别（如"水电维修"）
   - 选择"快速保修"或"自定义保修"
   - 填写报修信息：
     - 问题标题：`厨房水龙头漏水`
     - 问题描述：`厨房洗菜盆水龙头一直滴水，请师傅尽快修理`
     - 紧急程度：选择"中"
     - 上传现场照片（可选）
     - 填写联系电话
     - 选择预约时间
   - 点击"提交报修"

4. **验证提交结果**
   - 查看成功提示
   - 返回报修列表查看新提交的工单
   - 记录工单ID用于后续测试

### 第二步：管理员分配工单

1. **登录管理端**
   - 访问：`http://localhost:8081/api/swagger-ui.html`
   - 或使用Postman进行API测试

2. **获取未分配工单**
   ```bash
   curl -X GET "http://localhost:8081/api/workorder/list?status=PENDING" \
   -H "Authorization: Bearer {admin_token}"
   ```

3. **分配工单给维修工**
   ```bash
   curl -X PUT "http://localhost:8081/api/workorder/{workOrderId}/assign?assigneeId=3" \
   -H "Authorization: Bearer {admin_token}"
   ```

### 第三步：维修工接单处理

1. **维修工登录**
   - 在小程序中切换账号
   - 登录维修工账号：worker001 / 123456

2. **查看分配的工单**
   - 进入"工单"页面
   - 查看"待处理"tab中的工单

3. **接单开始处理**
   - 点击工单卡片查看详情
   - 点击"接单"按钮
   - 确认开始处理

4. **完成工单**
   - 点击"完成工单"
   - 输入维修费用（可选）
   - 确认完成

### 第四步：业主查看进度并评价

1. **切换回业主账号**
   - 重新登录业主账号

2. **查看工单状态**
   - 进入报修列表
   - 点击对应工单查看详情
   - 查看处理进度时间轴

3. **评价服务**
   - 工单完成后，点击"评价服务"
   - 选择评分（1-5星）
   - 填写服务反馈
   - 提交评价

## 🔍 API测试点

### 1. 工单提交API
```bash
curl -X POST "http://localhost:8081/api/workorder" \
-H "Content-Type: application/json" \
-H "Authorization: Bearer {token}" \
-d '{
  "title": "厨房水龙头漏水",
  "content": "厨房洗菜盆水龙头一直滴水",
  "category": "REPAIR",
  "priority": "MEDIUM",
  "submitterId": 1,
  "roomId": 1,
  "phone": "13800138001"
}'
```

### 2. 工单列表API
```bash
# 获取业主的工单
curl -X GET "http://localhost:8081/api/workorder/list?submitterId=1" \
-H "Authorization: Bearer {token}"

# 获取维修工的工单
curl -X GET "http://localhost:8081/api/workorder/list?assigneeId=3" \
-H "Authorization: Bearer {token}"
```

### 3. 工单详情API
```bash
curl -X GET "http://localhost:8081/api/workorder/{id}" \
-H "Authorization: Bearer {token}"
```

### 4. 工单操作API
```bash
# 开始处理
curl -X PUT "http://localhost:8081/api/workorder/{id}/start" \
-H "Authorization: Bearer {token}"

# 完成工单
curl -X PUT "http://localhost:8081/api/workorder/{id}/complete?cost=50" \
-H "Authorization: Bearer {token}"

# 评价工单
curl -X PUT "http://localhost:8081/api/workorder/{id}/rate?rating=5&feedback=服务很好" \
-H "Authorization: Bearer {token}"
```

## 🐛 常见问题

### 1. 工单提交失败
**可能原因**：
- 用户未登录或token过期
- 必填字段未填写
- roomId不存在

**解决方法**：
- 重新登录
- 检查表单验证
- 确认房屋数据存在

### 2. 图片上传失败
**可能原因**：
- 文件大小超限（>10MB）
- 文件格式不支持
- 上传目录权限问题

**解决方法**：
- 压缩图片
- 使用支持的格式（jpg/png/gif）
- 检查服务器目录权限

### 3. 工单状态不更新
**可能原因**：
- 权限不足
- 状态流转逻辑错误
- 数据库更新失败

**解决方法**：
- 检查用户权限
- 确认状态流转规则
- 查看后端日志
