## 🚀 使用方法

### 1. 启动后端服务
确保后端服务正常运行：
```bash
cd backend
start.bat  # 或 start-no-swagger.bat
```

### 2. 配置API地址
在 `uniapp/config/index.js` 中确认API地址：
```javascript
development: {
  baseURL: 'http://localhost:8081/api',
  debug: true
}
```

### 3. 测试连接
在小程序中访问测试页面：
- 路径：`pages/test/api-test`
- 运行"运行所有测试"验证API连接

### 4. 登录测试
使用以下测试账号登录：
- **管理员**: admin / 123456
- **业主**: owner001 / 123456
- **维修工**: worker001 / 123456
- **保安**: guard001 / 123456

## 🔄 数据流转

### 登录流程
```
1. 用户输入账号密码 → 
2. 调用 /auth/login → 
3. 保存token和用户信息 → 
4. 自动添加token到后续请求头
```

### 工单流程
```
业主提交工单 → POST /workorder →
管理员分配工单 → PUT /workorder/{id}/assign →
维修工接单 → PUT /workorder/{id}/start →
维修工完成 → PUT /workorder/{id}/complete →
业主评价 → PUT /workorder/{id}/rate
```

## 🔍 常见问题

### 1. API连接失败
**现象**: 小程序无法连接后端
**解决**: 
- 检查后端服务是否启动
- 确认API地址配置正确
- 使用API测试页面诊断

### 2. 登录失败
**现象**: 提示用户名或密码错误
**解决**:
- 使用预设的测试账号
- 检查后端数据库中的用户数据
- 查看后端日志确认请求到达

### 3. 工单数据为空
**现象**: 页面显示空数据
**解决**:
- 确认用户已登录
- 检查用户权限和数据关联
- 使用降级的模拟数据验证界面

### 4. 网络超时
**现象**: 请求超时错误
**解决**:
- 检查网络连接
- 调整timeout配置
- 使用本地网络环境测试
