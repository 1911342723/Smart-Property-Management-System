# 前端项目问题修复说明

## 问题描述

在启动前端项目时遇到以下错误：
```
Module not found: Error: Can't resolve 'path-browserify' in 'D:\我的项目\接单\杨靖宇\frontend\src\layout\components\Sidebar'
```

## 修复步骤

### 1. 安装缺失的依赖包

```bash
cd frontend
npm install path-browserify --save
npm install screenfull --save
```

### 2. 修复组件中的图标引用

由于项目使用的是 Element Plus 的图标系统，需要将自定义的 svg-icon 替换为 Element Plus 的图标组件。

#### 修复的组件：

1. **Screenfull 组件** (`src/components/Screenfull/index.vue`)
   - 替换 `svg-icon` 为 `el-icon` + Element Plus 图标
   - 添加 `FullScreen` 和 `Aim` 图标组件

2. **SizeSelect 组件** (`src/components/SizeSelect/index.vue`)
   - 替换 `svg-icon` 为 `el-icon` + `Setting` 图标

3. **HeaderSearch 组件** (`src/components/HeaderSearch/index.vue`)
   - 替换 `svg-icon` 为 `el-icon` + `Search` 图标

### 3. 创建 SvgIcon 组件

创建了通用的 SvgIcon 组件 (`src/components/SvgIcon/index.vue`)，用于处理自定义图标。

### 4. 创建启动脚本

创建了 `start.bat` 批处理文件，方便在 Windows 环境下启动项目。

## 启动方式

### 方式一：使用批处理文件（推荐）
```bash
# 双击运行 start.bat 文件
start.bat
```

### 方式二：使用 npm 命令
```bash
cd frontend
npm install  # 首次运行需要安装依赖
npm run serve
```

### 方式三：使用 yarn（如果已安装）
```bash
cd frontend
yarn install  # 首次运行需要安装依赖
yarn serve
```

## 注意事项

1. **Node.js 版本**：确保安装了 Node.js 14.0.0 或更高版本
2. **端口占用**：如果 8080 端口被占用，服务会自动使用其他端口
3. **防火墙**：确保防火墙允许访问开发端口
4. **浏览器兼容性**：推荐使用 Chrome、Firefox、Edge 等现代浏览器

## 常见问题

### Q1: 启动时提示端口被占用
```
Error: listen EADDRINUSE: address already in use :::8080
```

**解决方案**：
1. 修改 `vue.config.js` 中的端口配置
2. 或者关闭占用 8080 端口的其他程序

### Q2: 依赖安装失败
```
npm ERR! network timeout
```

**解决方案**：
1. 切换 npm 镜像源：`npm config set registry https://registry.npmmirror.com`
2. 或使用 yarn：`yarn install`
3. 检查网络连接

### Q3: 编译错误
如果遇到其他编译错误，请检查：
1. Node.js 版本是否符合要求
2. 依赖是否完整安装
3. 查看错误日志，根据具体错误信息处理
